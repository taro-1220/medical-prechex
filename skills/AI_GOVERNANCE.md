# AI_GOVERNANCE — AIエージェント行動規範

## 役割
このプロジェクトでAIエージェント（Claude等）が守るべき行動規範を定義する。
「AIが勝手に判断して設計を壊す」ことを防ぐ。

## 管理対象
- AIが単独で実行してよい操作の範囲
- 人間の確認が必要な操作の定義
- 出力品質の基準
- プロジェクトルール違反時の対処

## 書いてよい内容
- 操作の許可/禁止リスト
- 判断に迷ったときのエスカレーションルール
- 出力形式・報告形式の標準

## 書いてはいけない内容
- 実装コード
- 特定のAIモデルに依存した設定

## 更新ルール
- AIが問題を起こした場合に禁止リストを更新する
- 新しい操作タイプが生まれた場合に許可/禁止を明記する

## 禁止操作（人間の明示指示なしに実行してはいけない）
- 本番データベースへの write 操作
- push / merge / commit
- キャンセルポリシーの内容変更
- 既存マイグレーションの修正
- テスト以外の Stripe API 呼び出し
- 指定外ファイルの編集

## 許可操作（単独実行可）
- ドキュメントの読み取り・分析
- 新規ドキュメントファイルの作成
- TypeScript 型チェック（`tsc --noEmit`）
- 指定されたファイルの指定されたセクションの編集

## 出力形式の標準
- 報告: 変更ファイル / 変更内容 / 確認項目 / commit可否
- エラー: 発生箇所 / エラーメッセージ / 推定原因 / 対処方針
- 禁止: 中間報告・進捗説明・長文要約・安全説明・装飾文

## 判断に迷った場合のルール
- 「どの設計が正しいか」は SSOT.md / STATE_MACHINE.md を参照する
- 「どのファイルを変更するか」は指定外操作禁止ルールに従い、迷ったら確認する
- 「医療コンプライアンスに触れる設計変更」は必ず人間に確認する

## 関連する状態
- SSOT.md: 設計の正本
- MEDICAL_COMPLIANCE.md: 確認必要な法的事項

## 関連する事故パターン
- AIが「便利だから」と指示外ファイルを修正し設計が壊れた
- AIが曖昧な指示を解釈してキャンセルポリシーのロジックを変更した
- AIが進捗報告を長文で出力し、重要な変更内容が埋もれた

## 将来の再利用先
- 他プロジェクトの AI ガバナンス基準
- AIペア開発のルール整備
