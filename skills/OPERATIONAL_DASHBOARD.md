# OPERATIONAL_DASHBOARD — 運用ダッシュボード設計ガイド

## 役割
クリニックスタッフ・プラットフォーム運用者が日常業務で使うダッシュボードの設計方針を定義する。
「見落とし・誤操作・対応遅延」を設計で防ぐ。

## 管理対象
- ダッシュボードの表示要件
- 「要アクション」の定義とUI設計
- 操作の不可逆性の表示ルール
- 権限ごとの表示制御

## 書いてよい内容
- 表示すべき情報の優先順位と配置方針
- 「要アクション」の条件と表示期限
- 不可逆操作の確認ステップ設計
- ロール別の表示制御方針

## 書いてはいけない内容
- 実装コード・UIコンポーネント
- デザインカラー・アイコン定義
- 個別クリニックの運用ルール

## 更新ルール
- 「見落とし」や「誤操作」がインシデントになった場合に更新する
- 新しいステータスが STATE_MACHINE.md に追加された場合に更新する

## 要アクション定義

クリニックスタッフダッシュボードで「要アクション」として強調表示すべき状態:

| 状態 | アクション | 表示期限 |
|---|---|---|
| `authorization.expired` | カード再登録依頼 | 即時 |
| オーソリ失効6日前 | 注意表示（自動解消されない） | 失効まで |
| `visited`（capture 未実行） | capture 実行 | 来院確認から1時間 |
| `no_show` | キャンセル料回収確認 | 24時間以内 |
| `disputed` | 証拠確認・提出 | Stripe 期限前日まで |
| Webhook 失敗 | 手動確認（PF運用者向け） | 解消まで |

## 操作の不可逆性表示ルール
- 「来院確認」: 確認ダイアログ「この操作は取り消せません。来院を確認しましたか？」
- 「キャンセル料回収」: 金額と計算根拠を表示してから「請求する」
- 「no-show 取り消し」: 取り消し可能期間（24時間）を表示

## 権限別表示制御

| 機能 | owner | receptionist | doctor |
|---|---|---|---|
| 予約一覧 | ✓ | ✓ | ✓ |
| 来院確認 | ✓ | ✓ | — |
| キャンセル料回収 | ✓ | — | — |
| ポリシー変更 | ✓ | — | — |
| 売上レポート | ✓ | — | — |

## プラットフォーム運用者ダッシュボード
- DB-Stripe 乖離件数（0 でなければアラート）
- Webhook 失敗キュー（件数 + 最古レコード）
- 未処理 `no_show` 件数（24時間以上放置）
- 未処理 `disputed` 件数（対応期限3日前でアラート）

## 関連する状態
- STATE_MACHINE.md: 要アクション判定の基準
- OPERATIONS_OS.md: 定期確認タスクとの整合

## 関連する事故パターン
- `no_show` が一覧の下に埋もれ、24時間経過してキャンセル料回収できなかった
- 「来院確認」ボタンに確認ステップがなく、誤操作で未来院患者に請求された
- `receptionist` が誤ってキャンセル料回収操作を実行できた（権限設定ミス）
